# BankerDeck Workflow

## 1. Interpret the Topic

Convert the user topic into a banking-style mandate:

- Client/reader: investor, board, management, buyer, seller, lender, or internal strategy team.
- Decision: invest, acquire, sell, enter market, raise capital, approve budget, prioritize segment, or monitor industry.
- Output type: pitch book, investor presentation, industry primer, valuation deck, company profile, market scan, transaction memo.
- Geography, timeframe, company scope, and currency.
- Evidence required: public market data, financials, customer/industry data, transactions, expert commentary, benchmarks.

If any of these are unknown, make conservative assumptions and surface them in an assumptions slide or speaker note.

## 2. Create the Banker Storyline

Use this order unless the user asks for another structure:

1. Executive summary: 3-5 answer-first conclusions.
2. Context: why the topic matters now.
3. Market or company facts: size, growth, structure, economics.
4. Deep-dive analyses: customer, product, competition, financials, valuation, transactions, risks.
5. Strategic implications: what the evidence means.
6. Recommendation or next steps.
7. Appendix: data tables, methodology, backup charts.

## 3. Write Action Titles

Titles should read like:

- "AI data center demand is outpacing near-term power availability in Tier 1 Southeast Asian hubs"
- "Premium SaaS assets continue to trade at a scarcity multiple despite lower median software valuations"
- "China robotics supply chain has shifted from component availability to integration capability as the key bottleneck"

Avoid topic-only titles like "Market Overview" unless the slide is a divider.

## 4. Page Plan Template

For each slide, define:

- Slide number
- Action title
- Module type from `slide-modules.md`
- Main visual
- 3-6 supporting bullets or table rows
- Required data and formulas
- Sources and source-note text
- Open placeholders if data is unavailable

## 5. Quality Bar

Before finalizing, check:

- The first 3 pages can be read alone and still communicate the answer.
- Each chart has a clear conclusion in the title.
- Every number has a period, unit, and source.
- Design has banker-like density without clutter.
- Source notes are short but sufficient.
- All output elements remain editable.
